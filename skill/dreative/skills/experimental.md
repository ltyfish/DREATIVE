# Experimental

Use for one or two deliberate moments that change how the user perceives or
handles the product.

Novelty must be subject-grounded, legible, reversible where appropriate, and
surrounded by disciplined conventional structure. Prototype the riskiest
signature only when necessary.

Reject cursor toys, distortion for its own sake, random physics, hidden
navigation, and multiple unrelated experiments. If removing the novelty makes
the product clearer without losing meaning, the mechanism has not earned its
cost.

Use `../systems/GOLDEN_SYSTEMS.md` only after the concept identifies a useful
peak. If none of the bounded systems fits, build a focused project-native
prototype instead of searching a larger effect list.
