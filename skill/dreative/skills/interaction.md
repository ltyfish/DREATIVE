# Interaction

Use when input should change useful content or product state.

Every interaction needs a clear invitation, immediate response, meaningful
result, and equivalent pointer/touch/keyboard path. Prefer controls users can
understand over hidden gesture-only behavior.

State must survive rapid input without collisions or stale content. Keep
selected, loading, error, and completion feedback visible.

Reject cursor gimmicks, hover-only information, ornamental dragging, and
interaction that merely animates an unchanged card. Verify the real state
change in the browser.
